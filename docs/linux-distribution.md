# Linux Distribution Guide

Suwol Audio Reference 0.1.2 publishes Linux test builds as a zip of the Electron `linux-unpacked` directory. Tag releases also publish Linux AppImage and tar.gz assets with a signed checksum file. Automatic updates are supported only for packaged Linux AppImage builds.

## Current Artifact

- Linux zip: `release\Suwol.Audio.Reference.0.1.2.Linux.x64.zip`
- Linux AppImage: generated by electron-builder in `release/` on tag builds.
- Linux tar.gz: generated by electron-builder in `release/` on tag builds.
- Linux updater metadata: `latest-linux.yml` and generated `.blockmap` files from electron-builder.
- Signed checksum files in GitHub Releases: `checksums.txt`, `checksums.txt.asc`, and `suwol-release-public-key.asc`.
- Unpacked folder in CI: `release/linux-unpacked`

The Linux release artifacts are generated by GitHub Actions on `ubuntu-latest`:

```bash
npm run dist:linux:dir
npm run zip:linux
npm run check:release -- --platform linux
npm run dist:linux:release
npm run check:linux-updater -- dist --require-signature
```

Local Windows development should not try to force a Linux build. Let the GitHub Actions Linux job create and verify the Linux artifact.

## Running The Release Build

1. Download the Linux AppImage or tar.gz from the official GitHub Release. Zip artifacts remain available for zip smoke testing.
2. Download `checksums.txt`, `checksums.txt.asc`, and `suwol-release-public-key.asc`.
3. Verify the checksum signature and AppImage/tar.gz artifact hash before running:

```bash
gpg --import suwol-release-public-key.asc
gpg --verify checksums.txt.asc checksums.txt
sha256sum -c checksums.txt
```

4. For AppImage, set executable permission and run the downloaded file:

```bash
chmod +x ./Suwol*.AppImage
./Suwol*.AppImage
```

5. For tar.gz or zip, extract it to a writable folder and open a terminal in the extracted folder. Use `SHA256SUMS.txt` for zip-only hash verification.
6. If the executable bit is missing, run:

```bash
chmod +x "suwol-audio-reference"
```

7. Start the app:

```bash
./suwol-audio-reference
```

The exact executable name is checked by `scripts/check-release-artifacts.mjs`; it may be the package name or product-name-derived executable created by electron-builder.

## Runtime Notes

- AppImage and tar.gz are part of tag release distribution. deb/rpm are uploaded only if the existing electron-builder Linux configuration creates them.
- AppImage is the only Linux package type with automatic updates.
- tar.gz and zip builds use manual GitHub Releases downloads.
- Development mode never checks GitHub Releases for updates.
- The Settings Updates tab shows current version, distribution type, update status, manual check/download/install actions, expected release assets, checksum commands, and unsupported-package guidance.
- If an AppImage update is not detected, confirm the GitHub Release includes the AppImage, `latest-linux.yml`, and generated `.blockmap` files.
- Snap and Flatpak packages are not part of the 0.1.2 distribution.
- Playback support can vary by Linux distribution, Chromium codec support, and installed system libraries.
- Linux manual smoke should confirm launch, library open/create, one import, one playback attempt, Export Center preview, and app close.
- The app does not bundle an app-managed ffmpeg executable.
- The app does not provide audio editing, conversion, or transcoding.
- The app does not call AI, OpenAI, Codex, Whisper, LLM, cloud analysis, or cloud backup services.
- Generated QA fixture audio should not be published with release artifacts.

## CI Dependencies

The GitHub Actions Linux job installs the minimal packages needed for Electron packaging and zip creation:

```bash
sudo apt-get update
sudo apt-get install -y zip libnss3 libatk-bridge2.0-0 libgtk-3-0 libxss1
sudo apt-get install -y libasound2t64 || sudo apt-get install -y libasound2
```

If Ubuntu package names change, update `.github/workflows/build-release.yml` and keep this document aligned.

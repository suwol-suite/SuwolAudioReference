# Linux Distribution Guide

Suwol Audio Reference 0.1.1 publishes Linux test builds as a zip of the Electron `linux-unpacked` directory.

## Current Artifact

- Linux zip: `release\Suwol Audio Reference 0.1.1 Linux x64.zip`
- Unpacked folder in CI: `release/linux-unpacked`

The Linux zip is generated by GitHub Actions on `ubuntu-latest`:

```bash
npm run dist:linux:dir
npm run zip:linux
npm run check:release -- --platform linux
```

Local Windows development should not try to force a Linux build. Let the GitHub Actions Linux job create and verify the Linux artifact.

## Running The Zip Build

1. Download the Linux zip from the official GitHub Release.
2. If a checksum file is published, compare the zip SHA-256 hash before running.
3. Extract it to a writable folder.
4. Open a terminal in the extracted folder.
5. If the executable bit is missing, run:

```bash
chmod +x "suwol-audio-reference"
```

6. Start the app:

```bash
./suwol-audio-reference
```

The exact executable name is checked by `scripts/check-release-artifacts.mjs`; it may be the package name or product-name-derived executable created by electron-builder.

## Runtime Notes

- AppImage, deb, rpm, Snap, and Flatpak packages are not part of the 0.1.1 distribution.
- Playback support can vary by Linux distribution, Chromium codec support, and installed system libraries.
- Linux manual smoke should confirm launch, library open/create, one import, one playback attempt, Export Center preview, and app close.
- The app does not bundle an app-managed ffmpeg executable.
- The app does not provide audio editing, conversion, or transcoding.
- The app does not call AI, OpenAI, Codex, Whisper, LLM, cloud analysis, or cloud backup services.
- Generated QA fixture audio should not be published with release artifacts.

## CI Dependencies

The GitHub Actions Linux job installs the minimal packages needed for Electron packaging and zip creation:

```bash
sudo apt-get update
sudo apt-get install -y zip libnss3 libatk-bridge2.0-0 libgtk-3-0 libxss1
sudo apt-get install -y libasound2t64 || sudo apt-get install -y libasound2
```

If Ubuntu package names change, update `.github/workflows/build-release.yml` and keep this document aligned.
